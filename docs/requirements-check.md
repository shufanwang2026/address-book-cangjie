# 题目要求对照检查

题目名称：通讯录系统的设计与实现。

## 功能对照

| 题目要求 | 当前项目是否满足 | 对应实现 |
| --- | --- | --- |
| 录入通讯录信息 | 满足 | 菜单 `1. 录入联系人`，对应 `addContact` |
| 输入姓名删除记录 | 满足 | 菜单 `2. 删除联系人`，对应 `deleteContact` |
| 显示所有通讯录信息 | 满足 | 菜单 `3. 显示所有联系人`，对应 `listContacts` |
| 输入姓名查找联系人 | 满足 | 菜单 `4. 按姓名查找联系人`，对应 `searchContact` |
| 保存到指定磁盘文件 | 满足 | 菜单 `5. 保存到磁盘文件`，保存到 `contacts.txt` |
| 加载磁盘文件内容 | 满足 | 菜单 `6. 从磁盘文件加载`，从 `contacts.txt` 读取 |
| 退出通讯录系统 | 满足 | 菜单 `0. 退出系统` |
| 增删改功能 | 满足 | 增加、删除、修改分别对应菜单 `1`、`2`、`7` |

## 菜单编号

为了贴合题目描述，项目菜单编号调整为：

```text
1. 录入联系人
2. 删除联系人
3. 显示所有联系人
4. 按姓名查找联系人
5. 保存到磁盘文件
6. 从磁盘文件加载
7. 修改联系人
0. 退出系统
```

## 文件读写说明

程序使用 `contacts.txt` 作为数据文件。保存时将内存中的联系人写入文件；加载时读取文件内容并重新放入数组。

每一行是一条联系人记录，字段之间用制表符分隔：

```text
姓名    电话    邮箱    地址
```

这个格式比数据库简单，适合课程设计，也便于初学者理解“写文件”和“读文件”的过程。
